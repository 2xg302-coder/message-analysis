
### 4.3 时间范围过滤失效问题修复
- **问题描述**：用户反馈在 Data Explorer 页面选择“近一月”时间范围时，仅筛选出约 700 条数据，而数据库实际有 1700 条数据。
- **原因分析**：后端 SQL 查询使用 `date(created_at)` 函数进行日期过滤，但在某些环境下可能未能正确包含当天的全部数据（特别是带有时分秒的记录）。经排查，直接使用字符串比较（未包含当天结束时间）会导致当天（如 2026-03-06）生成的所有记录被排除。
- **解决方案**：
    - 修改 `server_py/services/news_service.py` 中的 `get_stats`, `get_tag_stats`, `get_type_stats`, `get_top_entities` 等方法。
    - 将 SQL 查询条件从 `date(created_at) BETWEEN ? AND ?` 改为显式的字符串时间范围比较：`created_at >= ? AND created_at <= ?`。
    - 在后端接收到的 `end_date`（如 `2026-03-06`）后自动追加 `T23:59:59.999999`，确保包含该日期的所有时刻记录。
    - 此修改修复了“当天数据被截断”的问题，确保统计结果与数据库实际记录一致。
