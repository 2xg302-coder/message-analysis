# 消息分析程序设计文档

> **注意**：本项目已完成从 Node.js 到 Python (FastAPI) 的重构。最新系统架构和使用说明请参考 [SYSTEM_INTRODUCTION.md](./SYSTEM_INTRODUCTION.md)。

## 1. 原始需求
1.  **收集消息**：从财经网站（如新浪 7x24）获取实时快讯。
2.  **后端存储**：设计后端存储海量消息。
3.  **智能处理**：
    *   数据清洗与去噪。
    *   价值评分与情感分析。
    *   LLM 深度推理（关联分析、趋势发现）。
4.  **可视化**：Web 界面展示新闻流、趋势图表和关注配置。

## 2. 当前实现 (Current Implementation)

### 2.1 架构概览
系统已拆分为独立的模块，采用 Python 微服务架构：

*   **后端 (Server)**: `server_py/`
    *   框架：FastAPI
    *   数据库：SQLite
    *   功能：数据采集 (Collectors)、预处理 (Processor)、API 服务、LLM 分析 (Analyzer)。
*   **前端 (Client)**: `client/`
    *   框架：React + Vite + Ant Design
    *   功能：实时新闻流 (快讯/深度模式)、数据可视化看板、事件追踪视图。

### 2.2 关键特性
1.  **双流采集**：支持快讯（Flash）和深度文章（Article）两种模式。
2.  **SimHash 去重**：使用局部敏感哈希算法进行文本去重。
3.  **FlashText NER**：高性能关键词提取，识别股票和实体。
4.  **混合分析**：结合规则引擎（实时响应）和 LLM（深度挖掘）进行情感和影响力评估。
5.  **事件追踪**：自动聚合相关新闻，形成事件发展时间轴。
